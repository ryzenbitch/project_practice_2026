## 3. Выполненные работы

### 3.1. Настройка Git и репозитория

Создан репозиторий на платформе GitHub: https://github.com/ryzenbitch/project_practice_2026

Освоены и применялись следующие команды и практики:
- `git init`, `git clone` — инициализация и клонирование;
- `git add`, `git commit` — фиксация изменений с осмысленными сообщениями;
- `git push`, `git pull` — синхронизация с удалённым репозиторием;
- `git branch`, `git checkout` — работа с ветками для разделения задач.

Репозиторий организован по структуре:
```
bot/       — исходный код Telegram-бота
site/      — статический сайт проекта
docs/      — документация проекта
reports/   — отчёты по практике
```

**Затраченное время:** ~5 часов.

### 3.2. Документация в Markdown

Все материалы проекта оформлены в формате Markdown. Подготовлены следующие документы:

- `README.md` — главная страница репозитория с описанием проекта;
- `docs/progress_log.md` — журнал прогресса с тремя записями;
- `reports/report.md` — итоговый отчёт по практике.

**Затраченное время:** ~5 часов.

### 3.3. Статический веб-сайт

Создан статический HTML/CSS сайт проекта Dark Machine. Сайт включает пять разделов:

- **Главная** — аннотация проекта, ключевые характеристики игры;
- **О проекте** — описание концепции, сеттинга, механик, биомов;
- **Участники** — состав команды и личный вклад каждого;
- **Журнал** — три записи о прогрессе разработки;
- **Ресурсы** — ссылки на инструменты и полезные материалы.

Сайт дополнен концепт-артами: иллюстрации главного персонажа, врага-паровой машины и фрагмента игрового мира.

**Затраченное время:** ~18 часов.

### 3.4. Разработка Telegram-бота (вариативная часть)

В рамках вариативной части задания разработан Telegram-бот проекта Dark Machine.

**Технологии:** Python 3, библиотека `python-telegram-bot`.

**Реализованный функционал:**
- `/start` — приветствие и краткое описание проекта;
- `/about` — подробная информация об игре Dark Machine;
- `/team` — состав команды и роли участников;

- `/feedback` — сбор пожеланий и обратной связи от пользователей.

Бот размещён в папке `bot/` репозитория, снабжён документацией и инструкцией по запуску в файле `bot/README.md`.

**Затраченное время:** ~35 часов.

## 4. Описание проекта Dark Machine

**Dark Machine** — пиксельная 2D метроидвания в сеттинге постапокалиптического будущего, где власть над миром полностью принадлежит машинам.

| Параметр | Значение |
|----------|----------|
| Жанр | Pixel Metroidvania |
| Формат | Singleplayer Indie Game |
| Камера | 2D Side View |
| Продолжительность | 5–10 часов |
| Игровой движок | Godot 4 |

### 4.1. Главный герой

Рыжеволосый выживший без чёткой гендерной принадлежности, одетый в лохмотья и тёмный плащ с капюшоном. Персонаж скрывается в тени и атакует из засады — воплощение последней человеческой воли к сопротивлению.

### 4.2. Мир игры

Мир Dark Machine разделён на три взаимосвязанных биома:
- **Ржавый квартал** — руины промышленного района, стартовая локация;
- **Нейронные катакомбы** — подземные тоннели под управлением ИИ;
- **Цитадель Ядра** — сердце машинной цивилизации, финальная зона.

Машины используют человеческие тела как биологическое топливо: золотистые резервуары с силуэтами людей внутри вмонтированы в стены фабрик.

### 4.3. Враги

- **Паровая тележка** — машина на колёсах с паровым котлом и массивным шипованным щитом. В режиме защиты щит поднимается вперёд, полностью закрывая корпус.

## 6. Итоги практики

### 6.1. Достигнутые результаты

В ходе практики команда:
- освоила базовые инструменты современной разработки: Git, Markdown, HTML/CSS;
- создала работающий Telegram-бот и статический сайт проекта;
- сформировала детальную документацию игрового проекта Dark Machine.

### 6.2. Трудности и их преодоление

Основная сложность — ограничение объёма проекта при богатой концепции. Решение: чёткое определение MVP и перенос дополнительных функций в roadmap.

Также потребовалось время на освоение совместной работы через Git: разрешение конфликтов слияния, соглашения о ветках. Практика показала ценность регулярных коммитов с осмысленными сообщениями.

### 6.3. Дальнейшие планы

- Создание играбельного прототипа первой локации в Godot 4;
- Разработка пиксельной графики: спрайты персонажа, врагов, тайловый набор;
- Расширение функциональности Telegram-бота.
